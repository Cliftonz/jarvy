# Ona Environments page

Headline: "A cloud computer for every developer and coding agent."

Stats claimed:
- <60s from zero to running environment
- 0% local compute used
- 24/7 coding agents working in parallel
- "400% productivity increase across our customers"

Pillars:
- Infrastructure for background agents
- Save time on environment setup
- Use your favorite tools (Source control, issue trackers, cloud, DBs, MCP)
- Ephemeral & isolated (every session fresh, destroyed after use)
- Enterprise security controls (VPC, command allowlists, egress control, scoped creds, audit logging)

Features:
- Flexible environment classes (Small, Regular, Large, GPU)
- Prebuilds + warm pools
- Full dev toolchain: devcontainer support, language servers, test runners, debuggers, DB connections

Pattern positioning ("agent has dev env" vs "sandbox as tool"):
- They claim Pattern 1 (full VM with dev container) — referencing Stripe + Ramp as users of this pattern
- Contrast to "sandbox as a tool" via API call

Notable post linked: "The last year of localhost" — argues background agents cannot run on a laptop.

Compliance icons: GDPR, SOC 2, Fortune 500, W3C.

Customer quote: Enterprise Architect at Kingland — "Our developers used to lose thousands of hours a year fixing broken environments. With Ona, that number is zero."

Talks referenced from: Stripe, Cloudflare, Uber, Harvey.
